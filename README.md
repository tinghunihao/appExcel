# 门店 Excel 智能处理工具

这是一个本地桌面应用，面向门店人员日常处理 Excel：

1. 地址行政区划智能补全：识别安徽省区县/地市，把省份和地级市补到地址最前端。
2. 动态字段提取与机型属性匹配：从原始表提取指定列，用 `69国标码` 匹配机型主数据表的 `69码`，合并机型属性。
3. A/B 表发货单号匹配：按电话、姓名、机型模糊包含关系，把 B 表发货单号填回 A 表。

## 运行方式

### macOS 当前环境

双击 `run_mac.command`，或在终端运行：

```bash
/Users/bixiav/.cache/codex-runtimes/codex-primary-runtime/dependencies/python/bin/python3 app/main.py
```

### Windows 门店电脑

如果门店电脑已安装 Python：

```bat
pip install -r requirements.txt
run_windows.bat
```

如果要发给门店一个单独的 exe，在 Windows 电脑上运行：

```bat
build_windows.bat
```

生成的程序在 `dist\门店Excel智能处理工具.exe`。

如果门店电脑是 Windows 32 位系统，需要使用 32 位 Python 打包：

```bat
build_windows_x86.bat
```

生成的程序在 `dist\门店Excel智能处理工具_x86.exe`。32 位老系统建议安装 32 位 Python 3.8。

如果 32 位老系统打开窗口版时报 `_tkinter` 相关错误，请改用兼容版：

```text
dist\门店Excel智能处理工具_x86_兼容版.exe
```

兼容版不依赖 Tkinter 窗口库，会用命令行菜单输入 Excel 路径和列名，Excel 处理逻辑与窗口版一致。

## Excel 要求

当前只处理 `.xlsx` 文件。

地址补全默认地址列名是 `详细地址`，可以在界面里修改。

字段提取与属性匹配默认配置：

| 配置 | 默认值 | 说明 |
|---|---|---|
| 提取列 | `1,3,5,8,10,69国标码` | 支持列序号或列名，逗号分隔 |
| 列名修改 | 空 | 格式如 `1=订单号,69国标码=商品条码` |
| 原始表69码列 | `69国标码` | 原始表中用于匹配的列 |
| 主数据69码列 | `69码` | 机型主数据表中用于匹配的列 |

匹配成功后会追加以下字段：

- `编码`
- `系统机型`
- `进货价`
- `厂家机型`
- `属性匹配状态`

A/B 表匹配默认列名：

| 表 | 字段 | 默认列名 |
|---|---|---|
| A表 | 电话 | 收货人电话 |
| A表 | 姓名 | 收货人姓名 |
| A表 | 厂家机型 | 厂家机型 |
| A表 | 模糊机型 | 系统机型 |
| B表 | 电话 | 收货人电话 |
| B表 | 姓名 | 收货人姓名 |
| B表 | 机型 | 机型 |
| B表 | 发货单号 | 发货单号 |

列名不一致时，可以在界面里直接改。

## 匹配规则

1. 电话会剔除空格、横杠等非数字字符。
2. 姓名会剔除空格。
3. 机型会统一大写、剔除空格，并把中文全角加号转成 `+`。
4. 优先用 A 表 `厂家机型` 与 B 表机型做精确匹配，忽略大小写和空格。
5. 精确匹配失败后，改用 A 表 `模糊机型` 字段与 B 表机型做模糊包含匹配。
6. 匹配失败时会在 A 表结果里写入：
   - `发货单号`
   - `匹配状态`
   - `匹配原因`

## 重要说明

地址知识库当前内置安徽省全部地级市及常见区县。若后续业务扩展到其他省，需要补充对应省份的区县到地市映射。
